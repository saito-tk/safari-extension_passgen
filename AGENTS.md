# リポジトリガイド

## プロジェクト概要
- このリポジトリは、Safari 向けのパスワードジェネレータ Web Extension を扱います。
- 現在の実装はポップアップ中心で、background script や content script は含みません。
- Safari のポップアップ制約を前提に、縦方向の情報量を抑えたタブ UI を維持してください。
- 後から Xcode で Safari App Extension に変換しやすいよう、拡張機能のソースは自己完結した構成を維持してください。

## 現在の UI 構成
- `設定` タブ
  - 文字数 / 件数
  - 文字セット選択
  - 記号選択
  - 記号文字列の貼り付け反映
  - 生成ルール
  - 表示テーマ切り替え
- `結果` タブ
  - 生成結果一覧
  - 進捗表示 `(n/件数)`
  - 行ごとのコピーアイコン

## プロジェクト構成
- `manifest.json`: Web Extension のマニフェスト。
- `popup.html`: ポップアップ UI のマークアップ。
- `popup.css`: テーマ、レイアウト、状態変化を含むスタイル。
- `popup.js`: パスワード生成ロジック、設定保存、テーマ切り替え、逐次生成、コピー処理。
- `README.md`: ユーザー向けのセットアップと仕様説明。
- `AGENTS.md`: Codex 向けの作業ガイド。

## プロダクト要件
- パスワード生成には暗号学的に安全な乱数を使います。生成ロジックでは `crypto.getRandomValues` を使い、`Math.random` は使わないでください。
- 基本 UX はポップアップ内で完結させてください。ユーザー要件がない限り、追加権限・background 処理・サイトアクセスは増やさないでください。
- UI 文言は、日本語対応の変更依頼がない限り日本語を維持してください。
- 設定保存は `storage.local` と互換性を保ってください。
- 結果一覧では長いパスワードを先頭 100 文字まで省略表示しますが、コピー時は全文字列を使ってください。
- テーマは見た目のみの設定です。生成ロジックへ影響を与えないでください。

## 入力仕様
- 文字数
  - 最小 `4`
  - 最大 `999999`
- 件数
  - 最小 `1`
  - 最大 `30`
  - ただし `min(30, floor(999999 / 文字数))` の動的上限を維持してください
- 数値入力は編集中に強制補正せず、フォーカスが外れた時点で補正してください。

## 記号選択の仕様
- `記号を含める` がオフの時は記号一覧と補助 UI を隠してください。
- 記号貼り付け欄に入力された文字列から、対応する記号だけを選択状態へ反映します。
- 対応外の文字だけが入力された場合は、記号選択状態を変えず、入力欄だけクリアしてください。
- `反映` ボタンは入力欄に何か入っている時だけ活性化してください。

## 生成中の仕様
- ポップアップを開いた時点では自動生成しません。`生成` ボタン押下時のみ生成します。
- 生成中は設定タブ側の入力項目を非活性にしてください。
- 生成中でも UI が固まりにくいよう、長い生成は分割して進めてください。
- 結果は逐次追加表示してください。

## テーマ仕様
- デフォルトは青テーマです。
- 少なくとも以下のテーマを維持してください。
  - 青
  - 緑
  - ピンク
  - 赤
  - 黄色
  - オレンジ
  - 紫
- テーマ選択 UI は設定タブ最下部の控えめな見た目の独立セクションに置いてください。
- プルダウンではなく色スウォッチで選ぶ現行 UI を維持してください。
- 選択状態やフォーカス色に青固定値を残さず、テーマ変数へ追従させてください。

## コーディング指針
- 素の JavaScript で、小さく明示的な関数を優先してください。ユーザーの指示がない限りフレームワークは導入しません。
- 明確な理由がない限り、ファイルは ASCII ベースを維持してください。
- DOM 操作は抽象化しすぎず、標準 API を優先してください。
- スタイル変更時は、現在の方向性であるガラス調パネル、軽い影、テーマ切り替え可能な色設計を崩さないでください。
- `manifest.json` には不要な権限を追加しないでください。

## 確認手順
- `popup.js` を変更したら `node --check popup.js` を実行してください。
- `manifest.json` を変更したら、JSON として壊れていないことを再確認してください。
- 手動確認では最低限以下を確認してください。
- ポップアップが正常に開くこと。
- `生成` ボタン押下時のみ生成されること。
- 生成中は設定系 UI が非活性になり、見た目も disabled と分かること。
- 選択した文字種に応じてパスワードが生成されること。
- 件数の動的上限が文字数に応じて変化すること。
- 記号貼り付け機能が仕様どおり動くこと。
- コピーアイコンが動作し、該当行にだけ `Copied!` が出ること。
- ポップアップを閉じて開き直しても設定とテーマが保持されること。

## Safari 向け注意点
- このリポジトリは Web Extension のソースであり、生成済み Xcode プロジェクトは含みません。
- Safari 向けのパッケージングが必要でも、明示的な依頼がない限り派生物の Xcode 成果物はコミットせず、変換元のソースを整える方を優先してください。

## 変更時のルール
- タスク範囲外のユーザー変更は巻き戻さないでください。
- セットアップ手順やユーザー向け挙動を変えた場合は `README.md` も更新してください。
- 新規ファイルを追加する場合も、明確な理由がない限りルート構成はシンプルに保ってください。
